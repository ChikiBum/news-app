# Фронтенд Новинної Аналітичної Платформи

## 🎯 Огляд Проекту

Комплексний React-застосунок для фронтенду новинної аналітичної та рекламної платформи. Застосунок забезпечує управління новинними стрічками в реальному часі, розширену аналітику даних, інтеграцію програматичної реклами та автентифікацію користувачів з сучасним, адаптивним дизайном на TypeScript та Tailwind CSS.

### Потік Валідації Даних

- **Клієнтська Валідація**: Zod схеми забезпечують валідацію в реальному часі
  - **Чому:** Миттєвий зворотний зв'язок користувача, зменшене навантаження на сервер, кращий UX
  - **Переваги:** Безпека типів, декларативні правила валідації, підтримка інтернаціоналізації
  - **Особливості:** Кастомні повідомлення про помилки, асинхронна валідація, композиція схем

- **Інтеграція React Hook Form**: Безшовне управління станом форм
  - **Чому:** Оптимізація продуктивності, мінімальні перерендери, легка інтеграція валідації
  - **Переваги:** Вбудована валідація, легка обробка помилок, функції доступності
  - **Особливості:** Валідація на рівні полів, обробка подання форм, можливості скидання

## 🛠 Основні Технології та Бібліотеки

### Фронтенд Фреймворк
- **React 19.1.1** - JavaScript бібліотека для створення користувацьких інтерфейсів
  - **Чому:** Компонентна архітектура, віртуальний DOM для продуктивності, відмінна екосистема
  - **Переваги:** Багаторазові компоненти, ефективні оновлення, сильна підтримка спільноти, конкурентні можливості
  - **Особливості:** Хуки для управління станом, context API для передачі пропсів, методи життєвого циклу, JSX синтаксис

### Інструменти Збірки та Розробки
- **Vite 6.3.6** - Інструмент збірки фронтенду нового покоління
  - **Чому:** Блискавично швидкий HMR, сучасні ES модулі, оптимізовані продакшн збірки
  - **Переваги:** Миттєвий старт сервера, швидкий досвід розробки, оптимізація tree-shaking
  - **Особливості:** Підтримка TypeScript, екосистема плагінів, dev сервер з HMR, аналіз бандлів

- **TypeScript ~5.8.3** - Типізований надмножина JavaScript
  - **Чому:** Безпека типів, краща підтримка IDE, раннє виявлення помилок
  - **Переваги:** Зменшені помилки виконання, кращий рефакторинг, покращений досвід розробника
  - **Особливості:** Статична перевірка типів, IntelliSense, визначення інтерфейсів, загальні типи

### Управління Станом
- **Zustand 5.0.8** - Легке рішення для управління станом
  - **Чому:** Простий API, мінімальний шаблонний код, відмінна підтримка TypeScript
  - **Переваги:** Не потрібні провайдери, підтримка devtools, екосистема middleware
  - **Особливості:** Інтеграція Immer, збереження, підписки, slice паттерн

### Отримання Даних та Серверний Стан
- **TanStack Query 5.87.4** - Потужна синхронізація даних для React
  - **Чому:** Автоматичне кешування, оновлення у фоні, оптимістичні оновлення
  - **Переваги:** Зменшений шаблонний код, автоматичне повторення помилок, офлайн підтримка
  - **Особливості:** Інвалідація запитів, обробка мутацій, безкінечні запити, паралельні запити

### Маршрутизація
- **React Router DOM 7.9.1** - Декларативна маршрутизація для React
  - **Чому:** Динамічна маршрутизація, вкладені маршрути, програматична навігація
  - **Переваги:** Розділення коду, охоронці маршрутів, обробка параметрів пошуку
  - **Особливості:** Ліниве завантаження, захищені маршрути, управління історією, link компоненти

### Управління Формами
- **React Hook Form 7.62.0** - Продуктивні форми з легкою валідацією
  - **Чому:** Мінімальні перерендери, вбудована валідація, легка інтеграція
  - **Переваги:** Краща продуктивність, менше коду, підтримка доступності

## 🔌 Інтеграція API

### Потік Автентифікації
```
Запит Клієнта → authApi → Backend → Відповідь → Auth Store → Оновлення UI
     ↓              ↓         ↓         ↓         ↓           ↓
Форма Входу → POST /auth/login → JWT Токен → Збереження Токена → Перенаправлення
```

### Шаблон Отримання Даних
```
Компонент → TanStack Query → API Модуль → Fetch → Кеш → Оновлення Компонента
    ↓           ↓              ↓          ↓      ↓         ↓
Сторінка Новин → useQuery → newsApi → GET /news → Кеш → Перерендер
```

#### gridApi.ts

### Етапи CI/CD Пайплайну

#### Етап 1: Забезпечення Якості Коду
- **Тригер**: Pull requests та push до main/master гілок
- **Призначення**: Забезпечення якості коду та запобігання поламаним збіркам
- **Інструменти**: Biome лінтер, TypeScript компілятор
- **Процес**:
  1. **Checkout Коду**: Отримання останнього стану репозиторію
  2. **Налаштування Середовища**: Інсталяція Node.js 20
  3. **Інсталяція Залежностей**: Інсталяція Biome лінтера
  4. **Лінтинг Коду**: Запуск всебічного аналізу коду
  5. **Перевірка Типів**: TypeScript компіляція та валідація
  6. **Перевірка Збірки**: Забезпечення успіху продакшн збірки

#### Етап 2: Автоматизований Деплоймент
- **Тригер**: Успішний push до main/master гілки
- **Призначення**: Деплоймент перевіреного коду в продакшн
- **Платформа**: Vercel хостинг платформа
- **Процес**:
  1. **Checkout Коду**: Свіжий клон репозиторію
  2. **Vercel Інтеграція**: Автоматизований тригер деплойменту
  3. **Процес Збірки**: Vite продакшн збірка
  4. **Оптимізація Ресурсів**: Оптимізація та стиснення бандлу
  5. **Глобальна Дистрибуція**: Деплоймент CDN через edge локації
  6. **Перевірки Здоров'я**: Верифікація після деплойменту

### Деталі Конфігурації CI/CD

#### Налаштування GitHub Actions
```yaml
# Конфігурація Робочого Процесу
name: CI/CD
on:
  pull_request:
    branches: [main, master]  # Валідація PR
  push:
    branches: [main, master]  # Продакшн деплоймент

jobs:
  biome-lint:                 # Завдання забезпечення якості
    name: Biome lint PR check
    runs-on: ubuntu-latest
    steps:
      - Checkout репозиторію
      - Налаштування Node.js середовища
      - Інсталяція залежностей розробки
      - Виконання процесу лінтингу
      - Валідація TypeScript компіляції
      
  deploy-vercel:             # Завдання деплойменту
    name: Deploy to Vercel
    needs: biome-lint        # Залежність від успіху лінтингу
    runs-on: ubuntu-latest
    if: github.event_name == 'push'  # Тільки для push подій
    steps:
      - Checkout репозиторію
      - Виконання Vercel деплойменту
```

### Інфраструктура Деплойменту

#### Інтеграція Vercel Платформи
- **Автоматичні Деплойменти**: Тригери деплойменту на основі Git
- **Попередні Деплойменти**: Середовища попереднього перегляду гілок та PR
- **Продакшн Деплойменти**: Автоматичний деплоймент головної гілки
- **Можливість Відкату**: Миттєвий відкат до попередніх версій
- **Управління Середовищем**: Безпечна обробка змінних середовища

### Стратегія Гілок та Робочі Процеси

#### Інтеграція Git Flow
- **Головна Гілка**: Готовий до продакшну код
- **Функціональні Гілки**: Розробка та тестування
- **Pull Requests**: Перегляд коду та валідація
- **Захищені Гілки**: Запобігання прямим push до main

#### Управління Релізами
- **Безперервний Деплоймент**: Автоматичні продакшн релізи
- **Попередні Середовища**: Тестові середовища на основі PR
- **Hotfix Деплоймент**: Можливість деплойменту термінових виправлень
- **Контроль Версій**: Управління версіями на основі Git
